## Computación Gráfica — Resumen (UI, UX, Web y Videojuegos) :contentReference[oaicite:0]{index=0}:contentReference[oaicite:1]{index=1}:contentReference[oaicite:2]{index=2}

### 1. Diseño de Interfaces (UI)
- Crear la **apariencia y disposición** de los elementos con los que interactúa el usuario: debe ser **intuitiva, atractiva y eficiente**.  
- Elementos clave: **diseño visual** (colores, tipografías, iconos), **layout** (botones, menús, formularios), **consistencia**, **interactividad** (animaciones) y **accesibilidad / responsive**.

### 2. Tipografía, color y herramientas
- Tipografía: usar **2–3 familias** como máximo; Google Fonts.  
- Color: elegir paleta coherente, revisar contraste (Contrast Checker).  
- Herramientas: **Wireframes** (Balsamiq, MockFlow); **Mockups** (Figma, Adobe XD, Illustrator); inspiración: Behance, Dribbble.

### 3. Diseño web y Front-End
- Diseño web: planificar y crear interfaces para sitios.  
- Front-end: HTML, CSS, JavaScript — lo que ve el usuario (tipografías, estilos, animaciones, adaptabilidad).  
- Maquetación: convertir diseño a archivos (HTML/CSS/JS) que el navegador renderiza en distintos dispositivos.

### 4. Aspectos prácticos (web)
- Optimizar tiempo de carga (imágenes, scripts).  
- Usabilidad: navegación clara.  
- Legibilidad: contraste, tamaño de letra, espaciado.

### 5. Experiencia de Usuario (UX)
- UX = percepción del usuario; meta: productos **útiles, usables y deseables**.  
- Proceso: investigación → análisis de flujos → wireframes → mockups → desarrollo → pruebas/retroalimentación → lanzamiento y monitoreo.

### 6. Videojuegos y motores gráficos
- Juego = software interactivo (PC, consolas, móviles, VR). Usa motores gráficos para render, físicas, audio y herramientas.  
- Motores: **Unity** (multiplataforma, amigable), **CryEngine** (alta calidad gráfica), **Unreal/UDK** (potente, buen soporte).

### 7. Diferencias clave
- **Lenguaje**: reglas y sintaxis (C++, Python).  
- **Librería**: funciones reutilizables que llamas.  
- **Framework**: estructura que controla flujo (p. ej. React).  
- **Motor gráfico**: framework especializado para juegos (Unity, Unreal).

## Introducción a HTML5 — Resumen (etiquetas, atributos, estructura, retos) :contentReference[oaicite:3]{index=3}

### 1. ¿Qué es HTML?
- **HTML (HyperText Markup Language)**: lenguaje de marcado para estructurar y desplegar páginas web.  
- No es lenguaje de programación (no ejecuta algoritmos).  
- Versión actual: **HTML5**.

### 2. Herramientas para crear sitios
- Editores: **Sublime Text**, **Visual Studio Code**.  
- Herramientas online: **CodePen**, **TutorialsPoint (online editor)**, **OnlineGDB**.

### 3. Estructura de etiquetas
- Formato: `<etiqueta>contenido</etiqueta>` — apertura, contenido (texto/otras etiquetas), cierre.  
- Buenas prácticas: etiquetas en **minúsculas**, usar solo etiquetas válidas de HTML5.

### 4. Atributos
- Añaden info y modifican comportamiento: `<etiqueta atributo="valor">`.  
- Usar **comillas dobles**.  
- Tipos:
  - **Conjunto finito**: solo valores permitidos.  
  - **Valores libres**: URL, texto.  
  - **Booleanos**: se escriben solos si son true (ej. `disabled`).

### 5. Atributos comunes
- `id` (único por página), `class` (reutilizable), `lang` (ISO 639-1), `translate` (yes|no), `title`, `style`.

### 6. Contenido y comentarios
- Contenido entre apertura y cierre; puede incluir otras etiquetas.  
- Comentarios: `<!-- comentario -->`.

### 7. `<div>` y semántica
- `<div>`: bloque genérico para estructurar y aplicar estilos.  
- HTML5 semántico: `<header>`, `<nav>`, `<section>`, `<article>`, `<footer>` — mejora accesibilidad y SEO.

### 8. Emmet
- Herramienta para generar rápidamente HTML/CSS (ej. `div*3` genera tres `<div>`).

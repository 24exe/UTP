# Computación Gráfica — Resumen de Clases (UI, UX, Web y Videojuegos)

## 1. Diseño de Interfaces (UI)
El diseño de interfaces se encarga de crear la apariencia visual y la disposición de los elementos con los que interactúa el usuario. Su objetivo principal es que la interfaz sea **intuitiva, atractiva y eficiente**.

### Elementos clave de la UI
- **Diseño visual**: colores, tipografías, iconos e imágenes.
- **Layout**: organización de botones, menús, formularios y secciones.
- **Consistencia**: mantener estilos y patrones visuales coherentes.
- **Interactividad**: animaciones, transiciones y respuestas visuales.
- **Accesibilidad y diseño responsivo**: adaptación a distintos dispositivos y usuarios.

---

## 2. Tipografía, Color y Herramientas de Diseño

### Tipografía
- Se recomienda usar **máximo 2 o 3 familias tipográficas**.
- Herramientas comunes: **Google Fonts**.

### Color
- Elegir una paleta coherente con la identidad visual.
- Verificar el contraste para asegurar legibilidad (ej. Contrast Checker).

### Herramientas de diseño
- **Wireframes** (estructura):
  - Balsamiq
  - MockFlow
- **Mockups** (diseño visual):
  - Figma
  - Adobe XD
  - Adobe Illustrator
- **Inspiración**:
  - Behance
  - Dribbble

---

## 3. Diseño Web y Front-End

### Diseño web
Proceso de planificación y creación de interfaces para sitios web, considerando estética, usabilidad y estructura.

### Front-end
Todo lo que el usuario ve e interactúa en el navegador:
- HTML
- CSS
- JavaScript

Incluye tipografías, estilos, animaciones, adaptabilidad y navegación.

### Maquetación
Transformar el diseño visual en archivos funcionales (HTML, CSS y JS) para que el navegador los renderice correctamente en diferentes dispositivos.

---

## 4. Aspectos Prácticos del Diseño Web
- **Tiempo de carga**: optimizar imágenes y scripts.
- **Usabilidad**: navegación clara y lógica.
- **Legibilidad**: buen contraste, tamaños adecuados y espaciado correcto.

---

## 5. Experiencia de Usuario (UX)

La UX se refiere a la **percepción y experiencia del usuario** al usar un producto digital. Su objetivo es crear productos **útiles, usables y agradables**.

### Proceso típico de UX
1. Investigación de usuarios y objetivos.
2. Análisis de la competencia y flujos de navegación.
3. Creación de wireframes.
4. Diseño de mockups.
5. Desarrollo del producto.
6. Retroalimentación (pruebas, encuestas, comentarios).
7. Lanzamiento y monitoreo.

---

## 6. Videojuegos y Motores Gráficos

Un videojuego es un software interactivo que puede ejecutarse en PC, consolas, dispositivos móviles o realidad virtual. Utiliza motores gráficos para gestionar gráficos, físicas, audio y lógica del juego.

### Motores gráficos mencionados
- **Unity**:
  - Multiplataforma.
  - Curva de aprendizaje amigable.
  - Muy usado en proyectos 2D y 3D.
- **CryEngine**:
  - Alta calidad gráfica.
  - Orientado a PC y consolas.
- **UDK (Unreal Development Kit)**:
  - Versión gratuita orientada a pequeños desarrolladores.
  - Soporte para juegos 2D y 3D.

---

## 7. Diferencias Clave: Lenguaje, Librería, Framework y Motor Gráfico

- **Lenguaje de programación**:
  - Define reglas y sintaxis para escribir código.
  - Ejemplos: C++, Python, JavaScript.

- **Librería**:
  - Conjunto de funciones reutilizables.
  - Tú decides cuándo y cómo usarlas.

- **Framework**:
  - Proporciona una estructura y controla el flujo del programa.
  - El desarrollador adapta su código al framework.

- **Motor gráfico**:
  - Framework especializado para videojuegos.
  - Incluye renderizado, físicas, audio, editor y herramientas de desarrollo.
  - Ejemplos: Unity, Unreal Engine.
